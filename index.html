Problema

Incompatibilidade de métodos HTTP → Erro 404

Por que isso acontece?

Seu formulário está tentando buscar dados (operação de leitura) usando GET, mas o webhook n8n está configurado para aceitar apenas POST.

Webhooks são normalmente configurados para POST porque:

  • São mais seguros para envio de dados
  • Permitem corpos de requisição maiores
  • São o padrão para APIs web modernas

Solução Recomendada

1 Alterar o formulário para usar POST

Modifique o código JavaScript do formulário para usar POST mesmo para operações de leitura:

// Código JavaScript corrigido async function buscarDadosPI(piNumber) { try { const response = await fetch('https://n8n.grupoom.com.br/webhook/CheckingForm', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ action: 'buscar_pi', n_pi: piNumber }) }); if (!response.ok) { throw new Error('Erro na resposta do servidor'); } return await response.json(); } catch (error) { console.error('Erro ao buscar PI:', error); throw error; } }

2 Verificar o nó "Identificar Ação"

Seu nó atual está correto para processar requisições POST:

// Seu código atual - correto para POST const body = $json.body || {}; if (body.action === 'buscar_pi') { return [{ json: { tipo: 'buscar_pi', n_pi: body.n_pi } }]; } else { return [{ json: { tipo: 'submissao', dados: $item.json } }]; }

3 Configuração de CORS

Seu nó "Responder ao Formulário" já está configurado com headers CORS adequados:

// Headers configurados no n8n Access-Control-Allow-Origin: * Access-Control-Allow-Methods: POST, OPTIONS Access-Control-Allow-Headers: Content-Type Content-Type: application/json

Vantagem desta abordagem: Mantém a consistência com as boas práticas de APIs web e a configuração atual do seu n8n.
